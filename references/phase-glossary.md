# Phase 7: Glossary — ユビキタス言語の整理

## 目的

全フェーズで蓄積された用語をユビキタス言語辞書として整理する。
Bounded Context ごとにスコープを明確にし、同じ用語が異なる意味を持つケースを記録する。

## 進行手順

### Step 1: 用語の収集

過去のフェーズ成果物 (event-storming.md, bounded-contexts.md, aggregates.md 等) から用語を抽出:
- Aggregate 名
- Entity 名
- Value Object 名
- Domain Event 名
- Command 名
- ビジネスルール内の概念

### Step 2: Bounded Context ごとの分類

各用語を所属する Bounded Context に分類。
同じ用語が複数のコンテキストで異なる意味を持つ場合、両方を記録。

### Step 3: 定義の精査

各用語について:
- 「この用語の定義をドメインエキスパートと開発者の両方が理解できるように書けますか?」
- 「この定義はコード上のクラス名/メソッド名と一致していますか?」
- 「曖昧さはありませんか?」

### Step 4: 不整合の発見

- コードの命名と用語集の不一致を検出
- チーム内で同じ概念に異なる名前が使われているケースを統一

## --analyze 時の追加手順

1. 既存コードのクラス名・メソッド名を抽出
2. 用語集と照合し、不一致を報告
3. 「コードでは `AutoAction` ですが用語集では `AutoMail` です。統一すべきですか?」
